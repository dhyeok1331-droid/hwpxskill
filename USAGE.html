📄 hwpxskill 사용법

한컴오피스 HWPX 문서를 AI 코딩 에이전트(Claude Code 등)에서 생성·편집·읽기 위한 스킬입니다. OWPML XML을 직접 다뤄서 python-hwpx API의 서식 버그를 피하고, 원본 문서의 구조를 거의 그대로 유지한 채 내용만 바꿀 수 있습니다.

XML-first 레퍼런스 복원 우선 페이지 드리프트 가드 Claude Code / Cursor / Codex CLI

1.설치

Claude Code

# 이 프로젝트에서만 사용
cp -r hwpxskill-main .claude/skills/hwpx

2.빠른 시작 — 가장 흔한 사용 패턴

대부분의 사용자는 스크립트를 직접 칠 필요가 없습니다. 채팅창에 .hwpx 파일을 첨부하고 "이 양식 그대로 두고 OOO 내용으로 바꿔줘"라고 요청하면, Claude가 아래 단계를 자동으로 수행합니다.

1
분석 — analyze_template.py로 원본의 header.xml / section0.xml을 추출
2
재작성 — 추출된 section XML을 복사해 구조는 유지하고 텍스트/데이터만 교체
3
빌드 — build_hwpx.py로 원본 header + 새 section을 조립
4
검증 — validate.py로 ZIP/XML 무결성 확인
5
페이지 가드 — page_guard.py로 원본 대비 쪽수 변동 여부 확인 (필수)
page_guard.py를 통과하지 못하면 완료로 처리하지 않고, 텍스트 분량을 줄이거나 구조를 다시 맞춰 재빌드합니다.

3.레퍼런스 기반 복원 (기본 워크플로우)

첨부된 HWPX가 있을 때 기본으로 적용되는 흐름입니다. 수동으로 실행할 경우 명령어는 다음과 같습니다.

# 1) 레퍼런스 분석 + XML 추출
python3 scripts/analyze_template.py reference.hwpx \
  --extract-header /tmp/ref_header.xml \
  --extract-section /tmp/ref_section.xml

# 2) /tmp/ref_section.xml을 복제해 /tmp/new_section0.xml 작성
#    (구조 유지, 텍스트/데이터만 요청에 맞게 수정)

# 3) 복원 빌드
python3 scripts/build_hwpx.py \
  --header /tmp/ref_header.xml \
  --section /tmp/new_section0.xml \
  --output result.hwpx

# 4) 검증
python3 scripts/validate.py result.hwpx

# 5) 쪽수 드리프트 가드 (필수)
python3 scripts/page_guard.py --reference reference.hwpx --output result.hwpx

99% 근접 복원 체크리스트

  • charPrIDRef, paraPrIDRef, borderFillIDRef 참조 체계 동일
  • 표의 rowCnt, colCnt, colSpan, rowSpan, cellSz, cellMargin 동일
  • 문단 순서·개수, 주요 빈 줄/구획 위치 동일
  • 페이지/여백/섹션(secPr) 동일
  • 변경은 사용자 요청 범위(본문 텍스트, 값, 항목명 등)로 제한

4.템플릿으로 새 문서 만들기

원본 파일이 없을 때, 내장 템플릿으로 바로 생성합니다.

# 빈 문서 (base 템플릿)
python3 scripts/build_hwpx.py --output result.hwpx

# 템플릿 지정
python3 scripts/build_hwpx.py --template gonmun --output result.hwpx

# section0.xml 오버라이드
python3 scripts/build_hwpx.py --template gonmun --section my_section0.xml --output result.hwpx

# 메타데이터 지정
python3 scripts/build_hwpx.py --template report --section my.xml \
  --title "제목" --creator "작성자" --output result.hwpx

5.기존 문서 직접 편집 (unpack → 수정 → pack)

# 1) HWPX → 디렉토리 (XML pretty-print)
python3 scripts/office/unpack.py document.hwpx ./unpacked/

# 2) ./unpacked/Contents/section0.xml (본문), header.xml (스타일) 직접 수정

# 3) 다시 HWPX로 패키징
python3 scripts/office/pack.py ./unpacked/ edited.hwpx

# 4) 검증
python3 scripts/validate.py edited.hwpx

6.텍스트 추출

# 순수 텍스트
python3 scripts/text_extract.py document.hwpx

# 표 포함
python3 scripts/text_extract.py document.hwpx --include-tables

# 마크다운 형식
python3 scripts/text_extract.py document.hwpx --format markdown

폴더 일괄 변환

여러 HWPX 파일을 한 폴더에 모아두고 한 번에 마크다운/텍스트로 바꿀 때 사용합니다.

# 폴더 안의 모든 .hwpx → 같은 폴더에 .md로
python3 scripts/batch_extract.py ./hwpx_files --format markdown

# 결과를 다른 폴더에 모아서 저장
python3 scripts/batch_extract.py ./hwpx_files --format markdown --output-dir ./md_out

# 하위 폴더까지 검색
python3 scripts/batch_extract.py ./hwpx_files --recursive

7.검증

python3 scripts/validate.py document.hwpx

검증 항목: ZIP 유효성, 필수 파일 존재, mimetype 내용/위치/압축방식, XML well-formedness.

꼭 알아야 할 주의사항

.hwp(바이너리) 미지원 — 옛 한글 바이너리 포맷은 지원하지 않습니다. 한글 오피스에서 "다른 이름으로 저장 → HWPX"로 변환 후 사용하세요.
쪽수 동일 필수 — 레퍼런스를 기반으로 작업할 때는 최종 결과의 쪽수가 원본과 같아야 합니다. validate.py 통과만으로는 완료가 아니며, page_guard.py까지 통과해야 완료로 간주합니다.
치환 우선 편집 — 사용자 요청이 없는 한 문단/표를 추가·삭제·분할·병합하지 않고, 기존 텍스트 노드를 교체하는 방식을 우선합니다.
  • secPr 필수: section0.xml 첫 문단의 첫 run에 반드시 secPr + colPr 포함
  • mimetype 순서: HWPX 패키징 시 mimetype은 첫 번째 ZIP 엔트리, ZIP_STORED 방식
  • 네임스페이스 보존: XML 편집 시 hp:, hs:, hh:, hc: 접두사 유지
  • itemCnt 정합성: header.xml의 charProperties/paraProperties/borderFills의 itemCnt가 실제 자식 수와 일치해야 함
  • ID 참조 정합성: section0.xml의 charPrIDRef/paraPrIDRef가 header.xml 정의와 일치해야 함
  • 빈 줄: <hp:t/> self-closing 태그 사용
